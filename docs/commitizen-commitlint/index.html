1 commitizen & cz-conventional-changelog 规范提交信息格式

1.1 安装、配置依赖

# git cz 替代 git commit，生成符合规范的 commit message。pnpm add commitizen -D# 一个符合 Angular 团队规范的 preset，按照我们指定的规范生成 commit message。pnpm add cz-conventional-changelog -D# 【全局模式】下, 需要 ~/.czrc 配置文件, 为 commitizen 指定 Adapter。echo '{ "path": "cz-conventional-changelog" }' > ~/.czrc# 【项目级配置】下# package.json{  "script": {    "commit": "git-cz",  },  "config": {    "commitizen": {      "path": "node_modules/cz-conventional-changelog"    }  }}

1.2 【可选】自定义 Adapter

# 指定一套符合自己团队的规范代替 Angular 规范pnpm add cz-customizable -D# package.json 指定配置{  "config": {    "commitizen": {      "path": "node_modules/cz-customizable"    }  }}# ===> 或者配置 .czrc 文件{ "path": "cz-customizable" }

2 commitlint 校验提交信息规范

# lint commit messagespnpm add @commitlint/cli -D# 一份符合 Angular 团队规范校验的配置pnpm add @commitlint/config-conventional -D# 对自定义的 Adapter 进行检测pnpm add commitlint-config-cz -D

2.1 配置 .commitlintrc

{  "extends": [    - "@commitlint/config-conventional" // 默认 commitizen adapter    - "cz" // 自定义 commitizen adapter    + "gitmoji"  ],  "rules": {}}

3【方法一】通过脚本生成 commit-msg 钩子

npx husky add .husky/commit-msg  'npx --no-install commitlint --edit "$1"'

3【方法二】在 package.json 中加入 commit-msg 钩子

"husky": {  "hooks": {    "commit-msg": "commitlint -E HUSKY_GIT_PARAMS"  }}
