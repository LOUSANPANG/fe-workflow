## 规范信息

### 规范提交信息

::tabs
---
tabs: [ 'bash', 'package.json' ]
---

#bash
```bash
# git cz 替代 git commit，生成符合规范的 commit message。
pnpm add -D commitizen

# 一个符合 Angular 团队规范的 preset，按照我们指定的规范生成 commit message。
pnpm add -D cz-conventional-changelog

# 【全局模式】下, 需要 ~/.czrc 配置文件, 为 commitizen 指定 Adapter。
echo '{ "path": "cz-conventional-changelog" }' > ~/.czrc
```

#package.json
```json
{
  "script": {
    "commit": "git-cz"
  },
  "config": {
    "commitizen": {
      "path": "node_modules/cz-conventional-changelog"
    }
  }
}
```

### 【可选】自定义提交规范

::tabs
---
tabs: [ 'bash', 'package.json', 'cz-config.js' ]
---

#bash
```bash
# 指定一套符合自己团队的规范代替 Angular 规范
pnpm add cz-customizable -D

# 【可选】gitmoji 头部添加了表情符号提交信息
pnpm add -D commitlint-config-gitmoji
```

#package.json
```json
{
  "config": {
    "commitizen": {
      "path": "node_modules/cz-customizable"
    }
  }
}
```

## 校验信息

### 校验提交信息

::tabs
---
tabs: [ 'bash', 'commitlintrc', 'package.json' ]
---

#bash
```bash
# lint commit messages
pnpm add @commitlint/cli -D

# 一份符合 Angular 团队规范校验的配置
pnpm add @commitlint/config-conventional -D

# 对自定义的 Adapter 进行检测
pnpm add commitlint-config-cz -D
```

#commitlintrc
```json
// .commitlintrc
{
  "extends": [
    - "@commitlint/config-conventional" // 默认 commitizen adapter
    - "cz" // 自定义 commitizen adapter
    + "gitmoji"
  ],
  "rules": {}
}
```

#package.json
```json
// 或者是通过脚本生成 commit-msg 钩子
// npx husky add .husky/commit-msg  'npx --no-install commitlint --edit "$1"'
{
  "husky": {
    "hooks": {
      "commit-msg": "commitlint -E HUSKY_GIT_PARAMS"
    }
  }
}
```
